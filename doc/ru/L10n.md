# Локализация

## Frontend

Для локализации ресурсов на клиенте используется `@ngx-translate/core`.

Файлы локализации модуля добавляются в папку `assets/i18n`. Имя файла должно соответствовать имени модуля и языку:

* `module-name.en.json`
* `module-name.ru.json`

Например, для модуля `weather-forecast-module` используются файлы:

* `projects/oip/src/assets/i18n/weather-forecast-module.en.json`
* `projects/oip/src/assets/i18n/weather-forecast-module.ru.json`

Пример структуры файла:

```json
{
  "weather-forecast-module": {
    "title": "Прогноз погоды",
    "content": {
      "table": {
        "date": "Дата",
        "temperatureC": "Температура, C",
        "temperatureF": "Температура, F",
        "summary": "Описание"
      }
    },
    "settings": {
      "title": "Настройки погоды",
      "dayCount": "Количество дней",
      "save": "Сохранить"
    },
    "errorFetchingMessage": "Ошибка получения данных о погоде:"
  }
}
```

### Использование в шаблоне

В шаблонах используется pipe `translate`. Компонент должен импортировать `TranslatePipe`.

```ts
import { TranslatePipe } from '@ngx-translate/core';

@Component({
  imports: [TranslatePipe],
  template: `
    <th pSortableColumn="date" scope="col">
      {{ 'weather-forecast-module.content.table.date' | translate }}
      <p-sortIcon field="date"/>
      <p-columnFilter display="menu" field="date" type="date"/>
    </th>

    <p-button
      icon="pi pi-save"
      [label]="'weather-forecast-module.settings.save' | translate"
      (onClick)="saveSettings(settings)"></p-button>
  `
})
export class WeatherForecastModuleComponent {}
```

### Использование в TypeScript

Компоненты модулей наследуются от `BaseModuleComponent<TBackendStoreSettings, TLocalStoreSettings>`. В базовом компоненте доступен метод `t`, который является оберткой над `TranslateService.instant`.

Метод удобно использовать в коде компонента, когда перевод нужен сразу, например для сообщения в обработчике ошибки.

```ts
export class WeatherForecastModuleComponent
  extends BaseModuleComponent<WeatherModuleSettings, WeatherModuleLocalSettings>
  implements OnInit, OnDestroy {
  protected override async onModuleInstanceChange(): Promise<void> {
    try {
      this.data = await this.dataService.getWeatherForecast({
        dayCount: this.settings.dayCount
      });
    } catch (error) {
      this.data = [];
      console.error(this.t('weather-forecast-module.errorFetchingMessage'), error);
      this.msgService.error(error.error.message);
    }
  }
}
```

Сигнатура метода:

```ts
public t(key: string | string[], interpolateParams?: InterpolationParameters): Translation | TranslationObject
```

Если в переводе используются параметры, их можно передать вторым аргументом:

```json
{
  "weather-forecast-module": {
    "errors": {
      "loadFailed": "Не удалось загрузить прогноз для {{dayCount}} дней"
    }
  }
}
```

```ts
this.t('weather-forecast-module.errors.loadFailed', { dayCount: this.settings.dayCount });
```

### Загрузка переводов

`BaseModuleComponent` получает имя текущего контроллера из маршрута и загружает переводы через `L10nService`.

```ts
this.route.url.subscribe((url) => {
  this.controller = url[0].path;
  this.l10n$ = this.l10nService.get(this.controller);
});
```

В компоненте модуля обычно не нужно вручную инжектить `L10nService`. Достаточно наследоваться от `BaseModuleComponent` и использовать:

* pipe `translate` в шаблоне;
* метод `this.t(...)` в TypeScript.

```ts
import { BaseModuleComponent } from 'oip-common';

export class WeatherForecastModuleComponent
  extends BaseModuleComponent<WeatherModuleSettings, WeatherModuleLocalSettings> {
}
```

После загрузки переводов ключи можно использовать в шаблоне через `translate`, а в коде компонента через `this.t(...)`.
